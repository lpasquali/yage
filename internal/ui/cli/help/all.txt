yage — bootstrap a Cluster API management plane and provision a workload cluster on any registered infrastructure provider

USAGE
    yage [OPTIONS]

DESCRIPTION
    Bootstraps a kind-based CAPI management plane (Cilium CNI: Ingress,
    kube-proxy replacement, Hubble) and provisions a workload cluster on
    the configured infrastructure provider — proxmox, aws, azure, gcp,
    hetzner, openstack, vsphere, docker, or any of the cost-only providers
    (digitalocean, linode, oci, ibmcloud). Pick one with --infra-provider
    or set INFRA_PROVIDER. By default, CAPI pivots to a managed mgmt
    cluster on the same provider (clusterctl move) before the workload
    is provisioned; --no-pivot keeps kind as the management plane.

OPTIONS

  General
    -f, --force                       Replace an existing kind cluster without prompting
        --no-delete-kind              Never delete the kind cluster (even with --force)
        --persist-local-secrets       Also write local CSI/proxmox-csi config files
    -p, --purge                       Delete generated files + Terraform state before continuing
    -b, --build-all                   Build and kind-load all images even when arm64 images exist online
        --regenerate-capi-manifest    Always re-run clusterctl to refresh the workload manifest
        --dry-run                     Print a structured plan of what every phase would do (based on
                                      the current config + env + kind state) and exit. No state is
                                      changed. Distinct from --pivot-dry-run (which actually
                                      provisions the mgmt cluster, stops at `clusterctl move`).
        --cost-compare                With --dry-run, print a side-by-side monthly-cost table for
                                      every registered provider (same logical cluster shape) using
                                      live FinOps APIs. Independent of the active infra-provider.
        --budget-usd-month F          With --dry-run, compute persistent-storage retention at the
                                      given monthly budget on the active provider. Internally still
                                      USD; displayed in the active taller currency.
        --print-pricing-setup VENDOR  Print the IAM/token setup CLI snippet for VENDOR's pricing API
                                      (aws, azure, gcp, hetzner, or "all") and exit. Same hint shown
                                      on first cost-related run when credentials are missing.
        --xapiri                      Launch the interactive configuration TUI and exit. Walks through
                                      every config knob, validates choices against the active provider,
                                      writes the result to disk for review. Currently a stub.
        --airgapped                   Disable every internet-requiring path: pricing fetchers, geo +
                                      FX detection, and the entire cloud-provider surface (AWS, Azure,
                                      GCP, Hetzner, DigitalOcean, Linode, OCI, IBM Cloud). Only on-prem
                                      providers (Proxmox, OpenStack, vSphere, Docker/CAPD) remain selectable.
                                      Env: YAGE_AIRGAPPED.
        --image-registry-mirror URL   Prefix passed to `clusterctl init` so CAPI provider images come
                                      from an internal mirror instead of the public registries
                                      (registry.k8s.io, ghcr.io). Required in airgapped deployments;
                                      yage warns when --airgapped is set without this. Format: host/path
                                      with no trailing slash. Env: YAGE_IMAGE_REGISTRY_MIRROR.
        --internal-ca-bundle PATH     PEM bundle of CAs trusted by every outbound HTTPS call yage makes
                                      (Helm pulls, OCI image pulls in clusterctl, kubectl-against-workload,
                                      pricing/inventory APIs). Loaded once at startup, installed on the Go
                                      HTTP transport, and exported as SSL_CERT_FILE so child processes
                                      (helm, clusterctl, kind, kubectl) trust it too. Useful in airgapped
                                      envs and behind corp TLS-intercepting proxies. Env: YAGE_INTERNAL_CA_BUNDLE.
        --helm-repo-mirror URL        Single base URL of an internal Helm chart repo (Harbor, ChartMuseum,
                                      …) that yage rewrites every chart-repo reference onto. Format:
                                      "https://harbor.internal/chartrepo/yage" (no trailing slash);
                                      yage rewrites e.g. "https://charts.jetstack.io/cert-manager" to
                                      "<mirror>/cert-manager". oci:// repos rewrite the same way. yage
                                      warns when --airgapped is set without this. Env: YAGE_HELM_REPO_MIRROR.
        --node-image IMAGE            Override the kind worker base image (kindest/node:vX.Y.Z) the
                                      management plane boots from. Useful when the public image is
                                      mirrored under a different name in an internal registry. Per-
                                      provider node templates / AMIs / Glance images stay on their
                                      own per-provider config fields. Env: YAGE_NODE_IMAGE.
        --hardware-cost-usd N         Self-hosted TCO (Proxmox / vSphere): full cluster capex in USD.
                                      Required to enable the cost estimate for self-hosted providers
                                      — without it they return ErrNotApplicable. Amortized straight-
                                      line over --hardware-useful-life-years.
        --hardware-useful-life-years N  Depreciation horizon for amortization (default 5).
        --hardware-watts N            Continuous power draw at typical load (NOT max nameplate).
                                      Drives the electricity opex line.
        --hardware-kwh-rate-usd N     Delivered electricity rate USD/kWh (default 0.15).
        --hardware-support-usd-month N  Flat monthly support / colo / vSphere licensing — added as-is.
        --resource-budget-fraction F  Cap the share of available Proxmox host CPU/memory/storage that
                                      the workload + (optional) management clusters may consume
                                      (default: 0.6667 — 2/3). The remaining ~1/3 is reserved for the
                                      host OS, hypervisor overhead, rollout slack, and the
                                      unknown-future workload that lands on the cluster after bootstrap.
        --bootstrap-mode MODE         Kubernetes flavor: kubeadm (default; full upstream Kubernetes,
                                      ~2 vCPU / 4 GiB per node) or k3s (single-binary, ~1 vCPU / 1 GiB
                                      per node — for resource-constrained envs). With k3s the
                                      orchestrator initializes CAPI with the K3s control-plane +
                                      bootstrap providers, and renders an embedded K3s flavor
                                      manifest in-process (no clusterctl generate cluster — upstream
                                      CAPMOX ships no k3s flavor; the template lives in
                                      internal/capimanifest/k3s_template.yaml).
        --aws-mode MODE               (--infrastructure-provider aws only) CAPA flavor:
                                        unmanaged   — self-managed K8s on EC2 (default).
                                        eks         — EKS-managed control plane (flat hourly fee +
                                                      EC2 Managed Node Group for workers; live-priced
                                                      via the AWS Bulk Pricing JSON).
                                        eks-fargate — EKS CP + Fargate workers (no EC2 fleet);
                                                      sized via --aws-fargate-pod-count /
                                                      --aws-fargate-pod-cpu / --aws-fargate-pod-memory-gib.
                                      EKS modes require --bootstrap-mode kubeadm (EKS is upstream
                                      Kubernetes managed by AWS; K3s isn't compatible).
                                      See docs/cost-and-pricing.md for tier examples.
        --aws-overhead-tier TIER      AWS service overhead bundle for the cost estimator:
                                        dev        — 0 NAT GWs, 1 ALB, 20 GB egress, 2 GB CloudWatch
                                        prod       — 1 NAT GW, 1 ALB, 100 GB egress, 10 GB CloudWatch (default)
                                        enterprise — 3 NAT GWs (multi-AZ HA), 2 ALBs, 1 NLB, 500 GB egress,
                                                     50 GB CloudWatch
        --aws-nat-gateway-count N     Override NAT GW count (priced live: hourly + per-GB processed).
        --aws-alb-count N             Override ALB count (priced live: hourly + LCU-hour).
        --aws-nlb-count N             Override NLB count (priced live: hourly + NLCU-hour).
        --aws-data-transfer-gb N      Internet egress estimate (live $/GB from AWS pricing).
        --aws-cloudwatch-logs-gb N    CloudWatch Logs ingested per month (live $/GB from AWS pricing).
        --azure-subscription-id ID    Azure subscription GUID CAPZ provisions into. Env: AZURE_SUBSCRIPTION_ID.
        --azure-tenant-id ID          Azure AD tenant GUID. Env: AZURE_TENANT_ID.

        --azure-client-id ID          Service-Principal / Managed-Identity / Workload-Identity client GUID.

                                      or workload-identity. Env: AZURE_IDENTITY_MODEL.
        --gcp-network-name NAME       VPC network the workload cluster's nodes live in.
                                      Empty = "default" (CAPG auto-mode VPC). Env: GCP_NETWORK_NAME.
        --gcp-image-family FAMILY     GCE image family for node OS (debian-12, ubuntu-2204-lts, ...).
                                      Empty = CAPG default. Env: GCP_IMAGE_FAMILY.
        --gcp-identity-model MODEL    GCP identity flavor: service-account (default;

        --openstack-cloud NAME        Named cloud entry in clouds.yaml (e.g. "devstack").

        --openstack-project-name NAME Keystone project / tenant. Env: OPENSTACK_PROJECT_NAME

        --openstack-image-name NAME   Glance image used for VM boot. Env: OPENSTACK_IMAGE_NAME.
        --openstack-control-plane-flavor FLAVOR  Nova flavor for control-plane VMs (e.g. m1.large).

        --openstack-worker-flavor FLAVOR  Nova flavor for worker VMs. Env: OPENSTACK_WORKER_FLAVOR.
        --openstack-dns-nameservers LIST  Comma-separated DNS servers pushed to subnet resources.

        --vsphere-datastore NAME      Datastore for VM disks. Env: VSPHERE_DATASTORE.

        --vsphere-tls-thumbprint HEX  vCenter cert thumbprint when self-signed.

        --vsphere-username USER       vCenter username. Env: VSPHERE_USERNAME.
        --vsphere-password PASS       vCenter password. Env: VSPHERE_PASSWORD.
        --allow-resource-overcommit   Downgrade the capacity check to a warning instead of failing
                                      the run when the plan exceeds --resource-budget-fraction.
        --overcommit-tolerance-pct N  How far above 100% of host capacity (existing VMs + planned
                                      cluster) is allowed before the orchestrator aborts. Default 15.
                                      Memory overcommit via Proxmox ballooning is fine within ~15%;
                                      beyond that, OOMs under load become likely. Verdicts:
                                        ≤ resource-budget-fraction          → ✓ proceed silently
                                        ≤ 1 + tolerance-pct/100             → ⚠  proceed with warning
                                        >  1 + tolerance-pct/100            → ❌ abort (unless
                                                                              --allow-resource-overcommit)

                                          2000 = 2 cores. The remainder of worker capacity is split
                                          equally across db / observability / product buckets.
        --system-apps-memory-mib N        Cluster-wide memory reserve for the same set; default 4096.

ENVIRONMENT

    Most flags have a matching upper-snake env var (e.g. --proxmox-url ↔ PROXMOX_URL).
    On a re-run against an existing kind cluster, the persisted Secret
    proxmox-bootstrap-system/proxmox-bootstrap-config takes precedence over env defaults
    for non-CLI fields. CLI flags marked _EXPLICIT win against the Secret merge:

        NODE_IP_RANGES           GATEWAY          IP_PREFIX
        DNS_SERVERS              ALLOWED_NODES
        WORKLOAD_CLUSTER_NAME    WORKLOAD_CLUSTER_NAMESPACE

    Pivot env vars (all default to empty/false unless noted):
        PIVOT_ENABLED            PIVOT_KEEP_KIND     PIVOT_DRY_RUN
        PIVOT_VERIFY_TIMEOUT     (default: 10m)
        MGMT_CLUSTER_NAME        (default: capi-management)
        MGMT_CLUSTER_NAMESPACE   (default: default)
        MGMT_KUBERNETES_VERSION  (default: WORKLOAD_KUBERNETES_VERSION)
        MGMT_CONTROL_PLANE_NUM_SOCKETS=1  MGMT_CONTROL_PLANE_NUM_CORES=2  MGMT_CONTROL_PLANE_MEMORY_MIB=4096
        MGMT_CONTROL_PLANE_MACHINE_COUNT=1   MGMT_WORKER_MACHINE_COUNT=0
        MGMT_CILIUM_HUBBLE=true              MGMT_CILIUM_LB_IPAM=false
        MGMT_PROXMOX_CSI_ENABLED=false       (mgmt is stateless by default)

    Per-machine-type Proxmox template overrides (empty → fall back to PROXMOX_TEMPLATE_ID):
        WORKLOAD_CONTROL_PLANE_TEMPLATE_ID   WORKLOAD_WORKER_TEMPLATE_ID
        MGMT_CONTROL_PLANE_TEMPLATE_ID       MGMT_WORKER_TEMPLATE_ID

EXAMPLES

    # See what a real run would do (no state changes)
    yage --dry-run

    # Standard bootstrap (kind = forever-management)
    yage \
        --proxmox-url https://pve.example:8006 \
        --admin-username root@pam!capi-bootstrap \
        --admin-token 11111111-2222-3333-4444-555555555555 \
        --region datacenter --node pve

    # Pivot an existing setup to a Proxmox-hosted management cluster
    # (workload keeps running; only the CAPI controller managing it moves).
    yage --pivot --pivot-keep-kind \
        --mgmt-control-plane-endpoint-ip 192.168.0.30 \
        --mgmt-node-ip-ranges 192.168.0.31-192.168.0.32

    # Validate the pivot before committing — provisions mgmt, runs
    # `clusterctl move --dry-run` and exits without moving state.
    yage --pivot --pivot-dry-run \
        --mgmt-control-plane-endpoint-ip 192.168.0.30 \
        --mgmt-node-ip-ranges 192.168.0.31-192.168.0.32

    # Per-machine-type templates (e.g. a hardened CP image, vanilla worker image)
    yage \
        --workload-control-plane-template-id 200 \
        --workload-worker-template-id 201

    # Snapshot kind state before a risky run
    yage --kind-backup ./kind-pre-pivot.tar.gz

    # Print Argo CD access info for the workload (no full bootstrap)
    yage --argocd-print-access
